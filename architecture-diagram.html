All compute (ALB, ECS, RDS, ElastiCache) sits in private subnets of a VPC; only the ALB and CloudFront are internet-facing. Each ops portal and customer surface deploys independently to its own S3 prefix/CloudFront behavior, so a marketing-site release never touches Café Ops. Swap Cognito in for the custom JWT auth in server/auth.ts if you want managed user pools instead of owning password storage.

2. Minimal — single web server (small scale / cheap start)

Users
↓
Nginx
TLS via Let's Encrypt · reverse proxy
↓
Static files
served directly by Nginx — all ui_kits/ frontend apps
Node API (PM2)
server/ — proxied at /api and /ws
Postgres
same VM or a managed instance (Neon, Supabase, RDS)

One EC2 instance (or any VPS — Lightsail, DigitalOcean, Hetzner): Nginx serves the static frontends and reverse-proxies /api + /ws to the Node process managed by PM2. Good for demos, pilots, or low-traffic launches — migrate to the AWS layout above when you need independent scaling, zero-downtime deploys, or Multi-AZ durability.
